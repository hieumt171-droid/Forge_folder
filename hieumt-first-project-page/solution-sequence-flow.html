Project Hub Sequence Flow

This document explains runtime flow for jira:projectPage with 3 sub-pages: Overview, Recent Issues, Team.

1) Startup flow

  1. Jira opens Project Hub page.
  2. Frontend bootstraps from src/frontend/index.jsx.
  3. Frontend calls view.getContext() to get project context.
  4. Frontend calls view.createHistory() and normalizes path to one of: /overview, /recent-issues, /team.
  5. React Router (HistoryRouter + Routes) renders the matched page.

2) Route-specific data flow

Overview route (/overview)

  • Frontend invoke: invoke('getOverviewData', { project })
  • Backend resolver runs JQL count query for project issues.
  • Returns: { projectKey, projectType, issueCount }

Recent Issues route (/recent-issues)

  • Frontend invoke: invoke('getRecentIssues', { project })
  • Backend runs JQL: project = KEY ORDER BY created DESC, maxResults=5.
  • Returns 5 newest issues with summary/status/priority/assignee/created.

Team route (/team)

  • Frontend invoke: invoke('getTeamAssignments', { project })
  • Backend runs JQL in 7 days window with non-empty assignee.
  • Returns de-duplicated assignee list.

3) F5 behavior and active sidebar

  • Because route is persisted by Forge history, pressing F5 on /recent-issues or /team re-renders the same route.
  • Manifest pages entries map route segments to Jira sidebar items, so active menu highlight follows current route.
  • Invalid routes are normalized to /overview in frontend startup.

4) Logging format (from formatLog)

Each resolver logs structured JSON at request start, response received, completion, and failure.
